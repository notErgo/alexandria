2.0Run Full Pipeline

Automated

Runs the following steps automatically for all active companies with a configured scraper:

  1. Probe each company IR source for reachability
  2. Enqueue live IR scrape jobs (via configured scraper mode: RSS, index page, etc.)
  3. Fetch new EDGAR filings (8-K, 10-Q, 10-K, 20-F) for all companies with a CIK
  4. Ingest IR press releases from the local archive folder
  5. Run LLM + regex extraction on all unextracted documents

LLM-directed crawl is included when the checkbox below is ticked — it runs after EDGAR and IR ingest, before extraction. Leave it unchecked for faster runs when crawl-sourced documents are not needed. After this pipeline completes, review disagreements in the Review tab.

Select companies

Button → Action Map

ButtonWhat it does
Acquire Archive Walks OffChain/Miner/Miner Monthly/, parses PDFs/HTMLs, inserts into reports. Skips existing unless force-flag is set.
Acquire IR Fetches live IR pages for all active companies using each company's configured scraper_mode. Stores raw text in reports.
Acquire EDGAR Calls SEC EDGAR full-text search for each company with a CIK. Fetches 8-K/10-Q/10-K (or 6-K/20-F for foreign filers). Deduplicates by accession number.
Auto-extract checkbox When checked, IR and EDGAR acquisition automatically run the extraction stage on all newly-ingested reports before marking the job complete.
Start Extraction Runs LLM + regex extraction on all unextracted reports (or one ticker). Writes agreed values to data_points and disagreements to review_queue.
Probe Targets Scraper governance: sends HTTP probes to IR URLs, checks RSS/index/template availability. Writes evidence to source_audit. Does not change modes.
Probe + Apply Modes Same as Probe Targets, then automatically updates scraper_mode for companies where evidence supports a mode change (skip → active only with explicit policy).
Sync Config Re-reads config/companies.json and upserts all company rows. Useful after editing the JSON file directly.
Scan Manifest Walks the archive directory and records every file in asset_manifest (inventory of what is on disk, before ingest). Run this after adding new files to the archive folder.
EDGAR Bridge Fills monthly coverage gaps by carrying forward or inferring values from quarterly/annual EDGAR filings when no monthly press release exists.

Key Concepts and Terminology

TermMeaning
reports The central ingestion table. One row per document (press release, EDGAR filing, archive file). Holds raw text, source type, and covering period. Extraction status tracked via extracted_at.
data_points Final extracted metric values. Populated when LLM and regex agree, or when an analyst approves a review item. These power the Explorer, Coverage, and Scorecard views.
review_queue Holds extraction candidates where LLM and regex disagree, or where only LLM found a value. Analyst approves, rejects, or edits each item. Approved items become data_points.
asset_manifest Inventory of files physically on disk in the archive folder. Separate from reports: a file can be in the manifest (known on disk) but not yet ingested (no reports row). Run Scan Manifest to populate. Tracks ingest_state: pending, ingested, legacy_undated.
covering_period The calendar month or quarter a report covers, in YYYY-MM or YYYY-Qn format. Inferred from filename, document body, or EDGAR filing metadata. Drives the coverage grid.
scraper_mode Controls how the IR scraper fetches live press releases for a company. One of: rss (parse RSS feed), index (scrape HTML listing page), template (generate URLs from a pattern), playwright (headless browser for JS-rendered pages), skip (do not scrape). EDGAR is a separate, always-on channel if a CIK is configured.
filing_regime Whether a company files with the SEC as a domestic (domestic) or foreign private issuer (foreign). Foreign issuers file 6-K/20-F/40-F instead of 8-K/10-K.
Agreement engine After extraction, compares the LLM value and regex value for each metric. If they agree within a configurable per-metric threshold (set in Metric Schema), the regex value goes directly to data_points. If they disagree or only LLM found a value, the item goes to review_queue.
Analyst protection Data points with extraction_method in (analyst, analyst_approved, review_approved, review_edited) are never overwritten by pipeline re-runs. Analyst corrections are permanent until explicitly changed.
Scraper governance Automated probing system that tests whether a company's configured IR source URLs are reachable and of the expected type. Records probe evidence in source_audit and flags companies whose configuration may be stale or misconfigured.
