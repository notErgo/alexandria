How this works — document to monthly slot pipeline

Three views, two data tracks. The toggle at the top of this panel switches between three views. The first two correspond to distinct data tracks that are kept separate in the database:

  • Monthly PRs / 8-K — monthly-granularity records (period format: YYYY-MM). These come from IR press releases, archive PDFs/HTMLs, and monthly production 8-K filings. Each document covers one calendar month. This is the primary view for month-by-month analysis.
  • SEC Filings (10-Q / 10-K) — quarterly/annual-granularity records (period format: YYYY-Qn or YYYY-FY). These come from 10-Q and 10-K filings. Each document covers an entire quarter or fiscal year. Values here are not visible in the Monthly view — they live in separate slots.
  • Interpret — manual LLM reprompt and per-report extraction controls.

Sources. Each document stream feeds one of the two tracks:

  • Monthly press releases (PR/IR) — filed by the company on its investor-relations page, typically within a week of month-end. Each document covers one calendar month. Goes to the Monthly track.
  • SEC 8-K filings (monthly production) — many miners also file monthly production results as an 8-K. The period is derived from the filing's period_of_report field (last day of production month), not the SEC submission date, so the data lands in the correct month. Goes to the Monthly track.
  • SEC 10-Q / 10-K filings — quarterly and annual reports. The pipeline anchors on the MD&A section (Item 2 for 10-Q, Item 7 for 10-K), skipping any Table-of-Contents entry with the same heading. Extracted values are stored against the quarter (e.g. "2024-Q3") or fiscal year (e.g. "2024-FY"). Goes to the SEC track only.

Stage 1 — Ingest (fetch and store). Documents are fetched and raw text is stored in the reports table. No extraction happens yet. The document is assigned a source_type (archive_html, archive_pdf, ir_press_release, edgar_8k, edgar_10q, edgar_10k).

Stage 2 — Keyword gate. Before any LLM call, the pipeline checks whether the document contains at least one mining-related anchor phrase (e.g. "bitcoin produced", "hash rate", "BTC mined"). Documents with no matching phrases are skipped and marked keyword_gated.

Stage 3 — LLM extraction. Documents that pass the gate are sent to the local Qwen LLM in a single batch call. The LLM returns a JSON object with one value per metric. A confidence score is computed per value. Values below the 0.75 threshold, or flagged as statistical outliers, are routed to the Review Queue. Values above the threshold are written directly to data_points as auto-accepted.

How 10-Q/10-K quarterly totals reach monthly slots — gap-fill. Because 10-Q and 10-K values are stored at quarterly/annual granularity, they do not appear in the Monthly view automatically. To propagate them into monthly slots, run the gap-fill step (Operations → Interpret tab → Gap Fill, or cli.py gap-fill --ticker TICKER). Gap-fill uses the quarterly total as a constraint and infers individual monthly values using delta inference for flow metrics (production, sales) or endpoint propagation for snapshot metrics (holdings). Inferred monthly values are written to the Monthly track with method badge D (derived) and are always overridden by a directly extracted monthly value if one exists.

Source priority within the Monthly track. When multiple documents contribute a value for the same month and metric, the highest-authority source wins:

  1. Analyst / finalized — manually confirmed or edited. Never overwritten by the pipeline.
  2. EDGAR (8-K) — auto-accepted from a monthly production 8-K (source priority 1).
  3. IR press release — auto-accepted from the company's investor-relations page (source priority 2).
  4. Archive — auto-accepted from a locally stored historical PDF or HTML (source priority 3).
  5. Derived (gap-fill) — inferred from a quarterly total (source priority 4).

If a value is in the Review Queue (not yet approved), the cell shows R. Approving it promotes it to data_points at the analyst tier.

Method badges. L = LLM auto-accepted, R = regex pattern match, A = analyst entry, F = finalized, D = derived (gap-fill), S = SEC filing, P = press release.

Coverage bar. Shows what fraction of expected monthly slots have at least one accepted value. Gaps are months where no document produced a confident value and no analyst has filled them in.

How to use this table
  • View/Edit toggle controls interaction. In View mode, left click does not mutate table state. In Edit mode, click a metric cell to edit it and use Add Row to create a manual month.
  • Editing a cell — click any metric cell in Edit mode to open an inline number input. Type a value, then press Enter or click OK to save it immediately. Press Escape to discard the change and restore the original value.
  • Switching to View mode with an open edit — clicking View (or any action that exits Edit mode) while a cell input is open automatically saves the typed value, equivalent to pressing OK. No data is lost.
  • Sync button — appears in Edit mode. Clicking it saves any open cell input and confirms all edits have been written to the database. Use it for peace of mind before navigating away.
  • Click a row in View mode — opens the document list directly below that row in the table.
  • Click a document in that row list — opens the review panel below the table for that specific source document.
  • Checkboxes only select rows for batch accept. Shift+click on checkboxes selects a range.
  • Hover over a value cell — browser tooltip shows the raw source snippet that produced the value.
  • Method badges next to values: R = regex, L = LLM, A = analyst, F = finalized, D = derived, S = SEC filing, P = pipeline.
  • Columns button — toggle individual metric columns on/off.
